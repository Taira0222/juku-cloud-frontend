# プロダクト要求定義書

## プロダクトビジョンと目的

### プロダクトビジョン
Juku Cloud Frontendは、塾管理業務のデジタル化を実現し、教師が生徒一人ひとりに最適な教育を提供できるよう支援するフロントエンドアプリケーションです。従来の紙ベースの管理から脱却し、授業記録、生徒情報、成績管理などを一元的に管理することで、教育の質を向上させることを目指します。

### 目的
- 教師の事務作業負担を軽減し、教育に集中できる時間を増やす
- 生徒の学習履歴と特性を可視化し、個別最適化された指導を実現する
- データに基づいた意思決定により、塾運営の効率化を図る

## ターゲットユーザーと課題・ニーズ

### ターゲットユーザー
| ロール | 説明 | 主な利用シーン |
|--------|------|---------------|
| 教師 | 授業を担当し、生徒を直接指導する | 授業記録作成、生徒情報確認、成績入力 |
| 管理者 | 塾全体の運営を管理する | ユーザー管理、データ分析、システム設定 |
| 生徒 | 授業を受講し、学習を進める | （将来的に）自分の学習履歴閲覧 |

### 解決したい課題
- **紙ベースの管理の非効率性**: 授業記録や生徒情報が紙で管理されており、検索や共有が困難
- **情報の分散**: 生徒情報が複数の場所に散在し、全体像の把握が難しい
- **データ活用の困難さ**: 過去のデータを分析して指導に活かすことができない
- **コミュニケーションの遅れ**: 教師間や保護者との情報共有に時間がかかる

### ユーザーニーズ
- 授業記録を素早く入力し、過去の記録を簡単に参照したい
- 生徒の特性や傾向を一目で把握したい
- モバイルデバイスからでもアクセスできるようにしたい
- 直感的で使いやすいUIが欲しい

## 主要な機能一覧

### 1. 授業管理機能
- 授業記録（LessonNote）の作成・編集・閲覧
- 授業内容、進捗状況、宿題の管理
- 生徒ごとの授業履歴表示

### 2. 生徒管理機能
- 生徒情報の登録・編集・閲覧
- 生徒特性（StudentTrait）の記録と管理
- 成績データの入力と推移グラフ表示

### 3. 教師管理機能
- 教師情報の登録・編集
- 担当クラス・生徒の割り当て
- 教師ごとの授業スケジュール表示

### 4. 分析・レポート機能
- 生徒の成績推移分析
- 授業実施状況のダッシュボード
- データエクスポート機能

### 5. ユーザー管理機能
- 認証・認可
- ロールベースのアクセス制御
- プロフィール管理

## 成功の定義

### KPI（重要業績評価指標）
- **業務効率化**: 授業記録作成時間を従来比50%削減
- **ユーザー満足度**: NPS（Net Promoter Score）≥40
- **システム利用率**: アクティブユーザー率≥80%
- **データ入力完了率**: 授業後24時間以内の記録入力率≥90%
- **システム応答時間**: ページロード時間≤2秒

### 成功条件
- [ ] 教師が授業記録を10分以内に完了できる
- [ ] 生徒情報を3クリック以内でアクセスできる
- [ ] モバイルデバイスで主要機能がストレスなく利用できる
- [ ] システムダウンタイム≤0.1%（年間）
- [ ] ユーザーからの高評価レビュー≥4.0/5.0

## ビジネス要件

### ビジネス目標
- 塾の運営効率を向上させ、より多くの生徒を受け入れられる体制を構築
- データドリブンな経営判断を可能にする
- 教師の働き方改革を支援し、優秀な人材の確保と定着を促進

### 収益モデル（想定）
- SaaS型の月額課金モデル
- 生徒数または教師数に応じた段階的な料金設定

### 市場要件
- 競合他社との差別化: より直感的なUI/UXと充実した分析機能
- 導入障壁の低減: 既存データの簡易インポート機能

## ユーザーストーリー

### 教師視点
**US-001**: 授業記録の迅速な作成
- **As a** 教師
- **I want to** 授業直後にスマートフォンから簡単に授業記録を作成したい
- **So that** 記憶が新鮮なうちに詳細な記録を残せる

**US-002**: 生徒特性の把握
- **As a** 教師
- **I want to** 授業前に生徒の特性や過去の記録を素早く確認したい
- **So that** 個々の生徒に合わせた指導ができる

**US-003**: 成績推移の可視化
- **As a** 教師
- **I want to** 生徒の成績推移をグラフで見たい
- **So that** 学習効果を定量的に評価できる

### 管理者視点
**US-004**: 塾全体の状況把握
- **As a** 管理者
- **I want to** ダッシュボードで塾全体の授業実施状況を一覧したい
- **So that** 運営上の問題を早期に発見できる

**US-005**: データ分析
- **As a** 管理者
- **I want to** 生徒の成績データや授業実施データをエクスポートしたい
- **So that** 詳細な分析や報告資料を作成できる

## 受け入れ条件

### AC-001: 授業記録作成（US-001関連）
- **GIVEN** 教師がログインしている
- **WHEN** 授業記録作成フォームに必要項目を入力して送信する
- **THEN** 記録が保存され、一覧画面に表示される
- **AND** 成功メッセージが表示される

### AC-002: 生徒特性閲覧（US-002関連）
- **GIVEN** 教師が生徒詳細画面を開いている
- **WHEN** 「特性」タブを選択する
- **THEN** 生徒の全ての特性がカテゴリ別に表示される
- **AND** 各特性の登録日時が確認できる

### AC-003: 成績グラフ表示（US-003関連）
- **GIVEN** 生徒の成績データが複数回登録されている
- **WHEN** 成績推移グラフを表示する
- **THEN** 時系列で成績の変化が折れ線グラフで表示される
- **AND** 科目ごとにフィルタリングできる

## 機能要件

### 必須機能（MVP: Minimum Viable Product）
1. ユーザー認証・認可
2. 授業記録のCRUD操作
3. 生徒情報のCRUD操作
4. 生徒特性の登録・表示
5. レスポンシブデザイン（PC・タブレット・スマートフォン対応）

### 優先度高
1. 成績管理機能
2. 教師管理機能
3. 検索・フィルタリング機能
4. ダッシュボード

### 優先度中
1. データエクスポート機能
2. 通知機能
3. コメント・メモ機能

### 優先度低（将来的に検討）
1. 保護者向けポータル
2. 生徒向け学習履歴閲覧
3. AIによる学習提案
4. カレンダー連携

## 非機能要件

### パフォーマンス
- ページロード時間: ≤2秒（初回）、≤1秒（キャッシュ利用時）
- API応答時間: ≤500ms（95パーセンタイル）
- 同時アクセス: 100ユーザーまで対応

### セキュリティ
- HTTPS通信の必須化
- 認証トークンの暗号化保存
- XSS、CSRF対策の実装
- ロールベースアクセス制御（RBAC）
- 個人情報の適切な取り扱い（GDPR、個人情報保護法準拠）

### アクセシビリティ
- WCAG 2.1 レベルAA準拠
- キーボード操作のサポート
- スクリーンリーダー対応
- カラーコントラスト比の確保

### 互換性
- **ブラウザ**: Chrome（最新版）、Firefox（最新版）、Safari（最新版）、Edge（最新版）
- **デバイス**: PC、タブレット、スマートフォン
- **OS**: Windows 10以降、macOS 10.15以降、iOS 14以降、Android 10以降

### 可用性
- 稼働率: ≥99.9%
- 計画メンテナンス: 月1回、深夜帯（事前通知必須）
- バックアップ: 日次自動バックアップ、30日間保持

### 保守性
- コードカバレッジ: ≥80%
- ドキュメント整備率: 100%（全機能）
- 技術的負債の定期的な解消

## 制約事項

### 技術的制約
- フロントエンドフレームワーク: React 19を使用
- バックエンド: Rails API（別リポジトリ）との連携が必須
- インフラ: 既存のクラウドインフラを利用

### ビジネス的制約
- 初期開発期間: 6ヶ月以内にMVPをリリース
- 予算制約: 開発リソースは限定的
- データ移行: 既存の紙ベースデータの移行サポートは限定的

### 法的・規制的制約
- 個人情報保護法の遵守
- 教育データの適切な管理と保護
- データの国外持ち出し禁止（日本国内のサーバーに保存）

### 運用上の制約
- サポート体制: 平日9:00-18:00のメールサポートのみ
- トレーニング: オンラインドキュメントとビデオチュートリアルで提供
- カスタマイズ: 標準機能のみ提供（個別カスタマイズは対応外）

## 前提条件

- ユーザーはインターネット接続環境がある
- バックエンドAPI（Rails）が稼働している
- ユーザーは基本的なPC/スマートフォン操作ができる
- ブラウザは最新版を使用している

## 依存関係

- バックエンドAPI（juku-cloud-backend）の開発進捗
- 認証サービスの選定と実装
- クラウドインフラの構築
